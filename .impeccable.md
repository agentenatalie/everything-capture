## Design Context

### Users
Personal knowledge management tool for a single power user. Used to capture, organize, and query saved content (articles, videos, social media posts) via browser extension, mobile web, and macOS desktop app.

### Brand Personality
Refined, minimal, professional — with warmth. Inspired by Apple system aesthetics, Notion's structured clarity, and ChatGPT's conversational fluidity.

### Aesthetic Direction
- **Visual tone**: Clean black-and-white dominant palette with subtle warm neutrals. Glass morphism effects for depth.
- **References**: Notion (structure, tool-feel), ChatGPT (conversational UI flow)
- **Anti-references**: Overly colorful dashboards, heavy gradients, gamified UIs
- **Theme**: Light mode only. Apple system fonts (SF Pro, PingFang SC).

### Design Principles
1. **Don't redesign — refine.** The current UI is intentionally designed and loved. Changes should only fix inconsistencies or polish details, never restructure.
2. **Color consistency over color theory.** When warm grays (#fbfbfa, #f5f5f4) and cold grays (#f8fafc, #f1f5f9) coexist, unify them within each visual context.
3. **Respect existing tokens.** Use CSS variables and existing color values. Don't introduce new palettes.
4. **Invisible improvements.** Good changes feel like the UI was always this way. No dramatic visual shifts.
5. **Density matters.** Compact where possible (e.g., stats line), spacious where it aids readability (e.g., AI chat).
