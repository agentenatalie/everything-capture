# Lessons

- 集成类问题不能只看配置和代码静态判断，必须在真实运行目录下分别复现 Notion、Obsidian、ZIP 三条链路，再给结论。
- 这个项目对启动目录敏感；凡是看到 `sqlite:///./items.db`、`static/...` 这类相对路径，优先排查 cwd 是否导致读写了错误的数据或文件。
- 前端视觉优化不能只统一一套玻璃效果；阅读详情、设置面板、列表行和卡片底部信息需要分别设计，优先保证对齐、焦点、层级和操作区固定，再谈质感。
- 列表不等于“把卡片纵向堆起来”；如果用户想要列表感，先保证容器是一体的、行与行之间有连续性，再在行级别加 hover 和选中反馈。
- 参考型 UI 改造不能只搬样式；像 Command K 这种组件要同时保留原站点已有的快捷键、加载反馈和核心动作，否则只是换皮不是适配。
- 这个项目的“网页详情页”有独立的图文顺序语义；改 UI 时不能只看容器和按钮，要保留 `content_blocks_json` / `canonical_html` / fallback 三层渲染优先级，否则图片会从正文里掉出去。
- 当用户明确要求“这个规则以后绝对不能再被动”，要落成仓库根目录的长期 MD 规范，而不是只写在临时任务记录里。
